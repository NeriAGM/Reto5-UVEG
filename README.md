
# Reto 5: Gestión de Repositorios con Git y GitHub

## Introducción
[cite_start]Este proyecto forma parte de las actividades de aprendizaje de la **Universidad Virtual del Estado de Guanajuato (UVEG)**. [cite_start]El objetivo principal es aplicar de manera práctica los conocimientos adquiridos sobre el uso de repositorios remotos y la gestión de código mediante la herramienta **Git**.

## ¿Por qué es importante este reto?
La realización de este ejercicio es fundamental para el desarrollo profesional de cualquier programador o entusiasta de la tecnología por las siguientes razones:

* [cite_start]**Dominio del flujo de trabajo real:** El reto simula un ciclo completo de trabajo, desde la creación de una cuenta y un repositorio hasta la implementación en "producción".
* [cite_start]**Gestión de errores y pruebas:** Al crear una rama específica para "pruebas", se aprende a experimentar con el código sin poner en riesgo la versión principal del proyecto.
* [cite_start]**Integración de cambios:** El proceso de unificar ramas (merge) permite comprender cómo se consolidan diferentes funcionalidades en un solo producto final[cite: 16, 58].
* [cite_start]**Cumplimiento de estándares:** Este reto no solo evalúa la capacidad técnica, sino también la habilidad para documentar correctamente y seguir normativas académicas y profesionales.

## Importancia de Git y GitHub en el entorno profesional

### Gestión de Versiones
[cite_start]**Git** es una herramienta esencial que permite llevar un registro detallado de cada modificación realizada en un proyecto. Esto es vital para:
1.  **Regresar en el tiempo:** Si un cambio genera un error, es posible volver a una versión anterior que funcionaba correctamente.
2.  **Organización:** Permite mantener un historial claro de quién hizo qué cambio y por qué razón.

### Trabajo Colaborativo
[cite_start]Por su parte, **GitHub** actúa como la plataforma en la nube que facilita el trabajo en equipo[cite: 17]. Su importancia radica en:
* **Sincronización:** Varios desarrolladores pueden trabajar en el mismo proyecto de forma simultánea desde diferentes partes del mundo.
* **Revisión de Código:** Permite que otros miembros del equipo revisen y sugieran mejoras antes de que los cambios se integren a la rama principal.
* [cite_start]**Visibilidad:** Funciona como un portafolio profesional donde se puede mostrar la calidad y el progreso del trabajo realizado a posibles reclutadores o asesores.

[cite_start]En conclusión, conocer y dominar estos comandos y plataformas no es solo un requisito académico, sino una competencia básica para cualquier persona que desee participar en el desarrollo de software moderno de manera eficiente y ordenada.
